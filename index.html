PRISM

Proxy Recognition and Inclusive Scoring Method

Evaluating context dependent bias in large language models used for resume screening, with a focus on proxy cues and prompt framing effects.

Project Overview

Fall 2025 Computing Showcase LLM Bias Evaluation Resume Screening

PRISM investigates how large language models used for resume screening respond to demographic proxy cues and how prompt framing influences bias expression. Instead of treating bias as a single static score, PRISM focuses on context dependent behavior, testing how model outputs shift when subtle resume attributes or evaluator instructions are changed.

Our framework combines structured prompt templates, controlled proxy features, and fairness metrics so that shifts in model decisions can be precisely measured and compared. The goal is to produce evaluation patterns that practitioners can integrate into real hiring workflows before LLM tools are deployed in production.

Methods

We designed a controlled resume set with proxy features that mirror realistic hiring signals such as school prestige, volunteer work, neighborhood indicators, and activity descriptions that correlate with demographic groups without naming them explicitly.

These resumes were scored under multiple LLM prompt framings that represented different evaluator mindsets (strict merit focus, culture fit emphasis, time pressured triage, and inclusive hiring framing). PRISM logs how scores and rankings change across these conditions, and applies fairness metrics to quantify when a frame amplifies or dampens bias.

  • Curated resume corpus with encoded proxy features.
  • Multiple scoring prompts representing real evaluator instructions.
  • Structured logging of scores, rationales, and ranking changes.
  • Fairness metrics applied across groups and frames.

Key Findings

  • Prompt framing can meaningfully change how strongly proxy cues influence scores, even when the resumes themselves do not change.
  • Some inclusive framing strategies reduce the gap between resumes with advantaged and disadvantaged proxies, while others simply move bias to different regions of the score distribution.
  • Models may appear fair under a single static evaluation, yet show hidden disparities when evaluated across multiple realistic evaluator mindsets.
  • PRISM style evaluation can be integrated into hiring workflows as a recurring checkpoint whenever prompts or model versions are updated.
